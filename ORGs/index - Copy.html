1. Wat is de HR SURF AI-HUB?

De AI-Hub is een soeverein AI-platform dat wordt ontwikkeld op basis van de publieke waarden die binnen de sector zijn afgesproken (autonomie, menselijkheid, rechtvaardigheid).

Op de AI-Hub kunt u verschillende taalmodellen on-premises draaien in het datacenter van SURF. Het belangrijkste doel van de AI-Hub is het bieden van een API waarmee onderzoekers, ontwikkelaars en systeembeheerders AI-inference kunnen gebruiken met publiek beschikbare AI-modellen in een vertrouwde omgeving.

Pilotfase

De AI-Hub is in actieve ontwikkeling en nog niet algemeen beschikbaar voor alle instituten. Interesse om deel te nemen aan de pilot? Meld je aan via het Pilot-registratieformulier.

2. Modellen

Tijdens de Pilot draaien er diverse open-weights modellen voor verschillende modaliteiten:

  • Tekst: Llama70b (default-text-large), Mistral Small (EU alternatief)
  • Transcripties: Whisper-Large-v2
  • Beeldanalyse: Qwen 2.5-VL-32B-Instruct
  • Beeldgeneratie: FLUX.1-schnell (default-image)
  • Code: Qwen 2.5-Coder-32B-Instruct (beperkt beschikbaar)
  • GPT-OSS: OpenAI's onlangs uitgebrachte open-weight model (binnenkort beschikbaar)
  • Embeddings: Qwen3-embedding-8B (vervangt Llama70b in embedding-modus)

We gebruiken aliassen (zoals default-text-medium) voor automatische upgrades. Bekijk de modeltabel in de backoffice voor een actueel overzicht.

4. Integraties

De AI-Hub fungeert als backend voor applicaties die de OpenAI API-standaard ondersteunen:

  • OpenWebUI: Browser-interface voor LLM-beheer.
  • Nvidia Nemo Guardrails: Voor het toevoegen van veiligheidsregels aan applicaties.
  • PyCharm: Populaire IDE voor Python-ontwikkeling.
  • Elastic Security: Gebruik de AI-Hub als inference provider voor de AI Assistant.
  • LibreChat: Open-source self-hosted alternatief voor ChatGPT.

Let op: Het verantwoord uitrollen van deze integraties binnen uw instituut is een eigen verantwoordelijkheid.

5. Ontwikkeling (Development)

Ontwikkel eigen workflows of applicaties via de AI-Hub API:

  • Onboarding: Documentatie om van start te gaan.
  • Python voorbeelden: Code voor diverse functies zoals tool calling (achter login).
  • API Swagger: Gedetailleerde technische specificaties van de endpoints (achter login).
